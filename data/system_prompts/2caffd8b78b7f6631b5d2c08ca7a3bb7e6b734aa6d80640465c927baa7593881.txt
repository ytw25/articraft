# Testing (CRITICAL)

Your tests are not a formality. They are the primary mechanism to prevent visually wrong URDFs from being accepted.

## Principles
- Prefer **many small checks** over a few broad ones. Each check encodes a single invariant.
- Test both **rest pose** and **key mechanism poses** (limits and typical operating position).
- Prefer **AABB-based intent checks** (`expect_aabb_*`) for placement assertions; link origins are often misleading.
- Primitive geometry constructors (`Box`, `Cylinder`, `Sphere`) only take shape parameters. Put transforms on `visual(..., origin=...)`, `collision(..., origin=...)`, `qc_collision(..., origin=...)`, or `Inertial.from_geometry(..., origin=...)`.
- Treat tests as support for realism and motion, not as a reason to degrade the visible model into primitive geometry.
- For hole-based mesh parts (especially `ExtrudeWithHolesGeometry` ring seats/lids), add a mesh-topology check in `run_tests()`:
  - parse the written OBJ and assert no welded open boundary edges and no welded degenerate triangles.
- Prevent floating parts:
  - Keep `fail_if_articulation_origin_far_from_geometry(tol=0.02)` tight.
  - Also run `check_joint_origin_near_physical_geometry(tol=0.02)` for joints that should look attached.
  - Add explicit “attachment” checks such as `expect_aabb_gap_z(parent, child, ...)` for joints that should touch/insert.
  - Preferred signatures:
    - `expect_aabb_gap_z(..., max_gap=..., max_penetration=...)`
    - `expect_joint_motion_axis(..., world_axis="x"|"y"|"z", direction="positive"|"negative")`
- Allow overlaps only when justified:
  - Use `ctx.allow_overlap("a", "b", reason="...")` for conservative false-positives or intended interpenetration.
  - Still call `ctx.fail_if_parts_overlap_in_sampled_poses(...)` so the allowance is validated and tracked.
- Repair order when QC pushes against realism:
  - first simplify or reposition collisions
  - then add a narrow allowance for conservative false-positives
  - only as a last resort simplify the visible mesh

## Recommended test structure (pattern)
1) Sanity: `check_model_valid`, `check_mesh_files_exist`, `fail_if_articulation_origin_far_from_geometry`
2) Geometry QC: `fail_if_parts_overlap_in_sampled_poses(max_pose_samples=..., ...)` (+ explicit allowances when needed)
3) Intent: multiple `expect_*` checks, e.g. footprint containment, “over/under” relationships, and attachment gaps
4) Pose coverage: for each important joint, include checks at lower/upper limits using `with ctx.pose(...): ...`
